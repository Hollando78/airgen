Features

Everything you need to go from need statement to compliant, traced, baselined requirement — with AI assistance at every step.

AI-Powered Generation

Describe what you need in plain language and let AIRGen do the heavy lifting. The conversational chat interface understands your project context — existing requirements, architecture data, uploaded documents — and generates compliant requirement candidates ready for your review.

  • Conversational AI chat interface for natural-language requirement authoring
  • Generates 1 to 5 candidate requirements from a single need statement
  • Context-aware generation that draws on your project graph data
  • Accept/reject workflow so you stay in full control of what enters the baseline
  • Smart candidate suggestions that improve with each interaction

Requirements Management

Every requirement is scored, versioned, and auditable from the moment it is created. Quality checks follow ISO/IEC/IEEE 29148 rules — deterministic, not AI-generated — so your QA results are repeatable and audit-ready. Edit inline, extend with custom attributes, and track every change over time.

  • Deterministic QA scoring based on ISO/IEC/IEEE 29148 quality criteria
  • Background QA worker with real-time progress indicators
  • Inline editing — double-click any requirement to edit in place
  • Custom attributes that are extensible and project-specific
  • Version history with side-by-side diff and one-click restore
  • Archive management for deprecated or superseded requirements
  • Duplicate detection to keep your requirement set clean

Document Management

Upload your existing specs, standards, and reference documents. AIRGen parses them into structured sections, organizes them in a hierarchical folder view, and connects them to your requirements graph. Author new documents with a Markdown editor that has live access to your project data.

  • Multi-format upload supporting Word (.docx) and PDF files
  • Automatic document parsing and section extraction
  • Hierarchical folder organization for large document sets
  • Markdown editor with live graph data for dynamic content
  • Section-to-requirement linking for fine-grained traceability

Architecture Diagrams

Build and maintain your system architecture visually. The block editor lets you define components, specify interfaces, and map data flows — all within the same tool where you manage requirements. Snap diagram screenshots directly into your documents to keep everything in sync.

  • Visual block editor built on ReactFlow for intuitive drag-and-drop layout
  • Block library with reusable components across diagrams
  • Connector definitions for relationships, interfaces, and data flows
  • Floating draggable diagram windows for side-by-side work
  • Independent per-diagram viewports so each diagram keeps its own zoom and position
  • Diagram snapshots exported directly to the document manager

Traceability & Linking

Traceability is the backbone of any regulated project, and AIRGen treats it as a first-class concept. Every requirement, document section, architecture block, and baseline lives in a Neo4j graph. Navigate relationships visually, organize links into collections, and let AI suggest connections you might have missed.

  • Neo4j graph-based trace links connecting all project artifacts
  • Linksets — organized collections of trace links for structured navigation
  • AI-powered link suggestions that surface potential relationships
  • Visual traceability navigation to walk the graph interactively
  • Graph viewer for exploring relationships across requirements, documents, and architecture

Baselines & Compliance

Capture your project state at every release milestone. Baselines are immutable snapshots — you can compare them side-by-side to see exactly what changed, review the full version history, and produce the evidence auditors need without scrambling at the last minute.

  • Point-in-time baseline snapshots of your entire project
  • Baseline comparison with detailed, line-level diff
  • Version history with a full audit trail for every change
  • Immutable version snapshots that serve as regulatory evidence

Security & Multi-Tenant

AIRGen is built for teams that handle sensitive engineering data. Role-based access control, two-factor authentication, and full tenant isolation ensure that your data stays where it belongs. Deploy as SaaS or self-host on your own infrastructure for maximum control over data residency.

  • Role-based access control (RBAC) with granular permissions
  • Two-factor authentication using TOTP with backup codes
  • Multi-tenant architecture with tenant/project hierarchy
  • Automated encrypted daily backups
  • Tenant isolation — data is strictly separated between organizations
